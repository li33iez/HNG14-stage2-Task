# HNG14-stage2-Task







Build and Deploy a Personal API



This stage focuses on developing a basic API and deploying it to a live server using proper DevOps practices. Building on Stage 0 (server provisioning and Nginx setup), you are required to create a minimal backend application with three endpoints (/, /health, /me) that return specific JSON responses with a 200 status code and fast response time (under 500ms).



The application must be tested locally and deployed on a VPS, running on a private port while being exposed publicly through an Nginx reverse proxy. Additionally, the service must be configured to run persistently using a process manager such as systemd or pm2.



Finally, all code must be pushed to a public GitHub repository with proper documentation, including setup instructions, endpoint details, and the live deployment URL. The project will be evaluated based on correctness, performance, deployment architecture, and documentation quality.
